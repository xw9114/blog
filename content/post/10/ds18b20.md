## 技能概述

掌握 DS18B20 数字温度传感器的底层协议通信及高精度数据解析。核心价值在于深入理解 **16 位补码存储机制** 与 **单总线（OneWire）时序控制**。能够解决从 12 位原始二进制数据到物理温度值的精确转换，特别是在零下低温环境（-55°C 至 125°C）下的符号位判定与数值还原，确保在蓝桥杯、工业监测等严苛场景下数据的准确性与稳定性。

## 核心能力矩阵

- **补码逆运算解析**：通过 **“取反加一”** 的逻辑，将 DS18B20 高位符号扩展的负值原始码流还原为绝对值，彻底解决负温显示为巨大正数的逻辑溢出问题。
- **高精度位权换算**：精通 12 位分辨率下 $0.0625^{\circ}C$ 的位权计算，利用浮点数或定点数优化技巧，实现微秒级响应下的数据缩放与精度保留。
- **单总线时序精控**：严格遵循 `Init-Write-Read` 序列，精确控制 $\mu s$ 级延时，确保存储器操作（ROM Command）与功能命令（Convert T）的可靠执行。
- **UI 映射与数码管驱动**：实现数值与显示逻辑的解耦，支持负号自动判定、消隐处理及带小数点位显示的动态刷新方案。

## 深度理论：16 位补码结构解析

在单片机处理 DS18B20 零下温度或传感器偏移量时，理解 16 位补码（Two's Complement）是规避“数据爆炸”的关键。

### 1. 结构拆解
16 位补码将 Bit 15 到 Bit 0 划分为符号位和数值位：
* **符号位 (Sign Bit) - 第 15 位**：最高位（MSB）决定正负。`0` 表示正数或零，`1` 表示负数。
* **数值位 - 第 0 到 14 位**：正数直接存放原码；负数存放其绝对值**“取反加一”**后的结果。

### 2. 为什么 DS18B20 读出负温是 0xF8xx？
当温度为负时，DS18B20 会进行**符号扩展**。以 -0.5°C 为例：
1.  **绝对值原码**：0.5 对应的原始数值是 `0000 0000 0000 1000` (8 * 0.0625 = 0.5)。
2.  **求补码（负数）**：
    * 取反：`1111 1111 1111 0111`
    * 加一：`1111 1111 1111 1000` → 十六进制即为 **0xFFF8**。

### 3. 换算逻辑映射表
| 数据特征 | 判断方式 | 处理方法 |
| :--- | :--- | :--- |
| **最高位为 0** | `(dat & 0x8000) == 0` | **正数**：直接使用。 |
| **最高位为 1** | `(dat & 0x8000) != 0` | **负数**：执行 `~dat + 1` 提取绝对值，计算后补负号。 |

> **关键提醒**：在 C 语言中定义 `unsigned int` 会使编译器无视符号位，将第 15 位误判为数值（32768），这也是导致负温显示为数千度的根本原因。

## 代码能力展现

以下代码展示了在底层驱动中利用补码逻辑处理负温，以及在应用层如何安全显示。

```c
/* 核心逻辑：底层温度读取与补码还原 */
float Read_Temperature_Safe(void) {
    unsigned char low, high;
    unsigned int raw;
    float result;
    bit is_negative = 0;

    init_ds18b20();
    Write_DS18B20(0xCC); // Skip ROM
    Write_DS18B20(0x44); // Convert T
    Delay_OneWire(200);

    init_ds18b20();
    Write_DS18B20(0xCC);
    Write_DS18B20(0xBE); // Read Scratchpad
    
    low = Read_DS18B20();
    high = Read_DS18B20();
    
    // 合并数据：高 5 位为符号位 SSSSSGGG GGGGGFFF
    raw = (high << 8) | low;

    // 负温判定
    if ((raw & 0xF800) != 0) {
        is_negative = 1;
        // 核心魔法：补码转原码绝对值
        raw = ~raw + 1;
    }
    
    result = raw * 6.25f; // 放大 100 倍以保留两位小数精度
    return is_negative ? -result : result;
}

/* 应用层：数码管显示映射逻辑 */
void Display_Temperature(float temp) {
    unsigned int abs_val;
    
    if (temp < 0) {
        buffer[0] = 17; // 对应段码表中的 '-'
        abs_val = (unsigned int)(-temp);
    } else {
        buffer[0] = 25; 
        abs_val = (unsigned int)temp;
    }

    // 提取数位 (假设 abs_val = 2512 即 25.12°C)
    buffer[1] = abs_val / 1000;             // 十位
    buffer[2] = (abs_val / 100 % 10) | 0x80; // 个位带小数点
    buffer[3] = abs_val / 10 % 10;          // 十分位
    buffer[4] = abs_val % 10;               // 百分位
}
