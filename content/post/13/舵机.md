## 技能概述

掌握伺服控制系统（Servo System）的核心底层逻辑，能够从物理感知、信号通信、软件映射到硬件执行四个维度构建完整的控制闭环。不仅理解 PWM（脉冲宽度调制）的表面协议，更洞察其背后的时间片分配、数字域映射原理以及闭环系统的动态平衡哲学。该技能适用于多自由度机械臂设计、无人机云台控制及工业自动化精密定位场景，实现“指哪打哪”的物理交互精度。

## 核心底层概念解析

* **闭环系统的“动态平衡”哲学**：普通直流电机是“开环”的盲汉，而舵机是具备“自省”能力的系统。其核心概念在于**误差驱动（Error-Driven）**：系统通过电位器（位置传感器）实时采集当前机械角度（反馈量），与 PWM 解析出的目标角度（给定量）进行求差运算。只要误差不为零，内部控制器就会输出动力去“抹平”这个误差。这种负反馈机制是机械臂对抗重力、摩擦力等外部扰动的根本保障。
* **时域通信的抗干扰哲学**：为什么单片机不直接给舵机输出变化的模拟电压？因为在复杂的机器人走线中，电压会随线缆长度和环境干扰产生不可控的压降。PWM 协议的巧妙之处在于它将**“空间信息（角度）”编码为“时间信息（脉宽）”**。在 **20ms** 的周期内，高电平持续的绝对时间（**0.5ms ~ 2.5ms**）才是有效载荷。这种只看“时间阈值”不看“电压振幅”的数字信号，在物理层面上具备极强的抗噪能力。
* **数字世界与物理世界的时基映射**：微控制器（MCU）没有现实时间的感知能力，只有“时钟节拍（Tick）”。定时器预分频器（PSC）决定了每一次计时的“刻度大小”，而重装载寄存器（ARR）决定了 PWM 的“总周期”。改变捕获比较寄存器（CCR）的值，本质上是在调整一个周期内高低电平的配比。将物理学的秒与度，精确翻译为芯片底层的寄存器数值，是连接软件算法与物理执行的桥梁。
* **模拟计算的暴力美学（误差脉冲提取）**：在早期的模拟舵机内部，误差的计算不依赖数字代码，而是依靠纯粹的物理电路机制——“双轨赛跑”。外部指令脉冲与内部电位器生成的反馈脉冲同时触发，并送入硬件逻辑比较器。两个脉冲等宽时，不输出电流；一旦存在时间差，多出的那部分时间就会形成一个“误差脉冲”。这个微小的脉冲通过电容积分电路被拉伸放大，稳稳推开 H 桥的大门，转化为驱动电机的强力直流电压。这是不用一行代码实现“精准相减”的硬件艺术。

## 代码能力展现

以下为基于 STM32 HAL 库编写的舵机控制核心算法。该代码通过精确的数学映射，将抽象的角度值转化为定时器底层的比较寄存器数值（CCR）。

```c
/**
 * @brief  将指定角度转换为定时器 PWM 比较值 (CCR)
 * @note   假设定时器预分频后频率为 100kHz (10us/tick), PWM 周期为 20ms (ARR=2000)
 * 映射关系: 0.5ms (50 counts) -> 0°, 2.5ms (250 counts) -> 180°
 * @param  angle: 目标角度 (0.0 - 180.0)
 * @return uint32_t: 定时器比较值
 */
uint32_t Servo_Angle_To_Compare(float angle) {
    // 约束输入范围，防止机械撞墙卡死，引发电机过载烧毁
    if (angle < 0.0f)   angle = 0.0f;
    if (angle > 180.0f) angle = 180.0f;

    /* 线性映射公式推导:
     * 概念核心：将 [0, 180] 的角度空间，线性映射到 [50, 250] 的定时器计数值空间
     * CCR = Offset + (Angle / MaxAngle) * Range
     * CCR = 50 + (angle / 180.0) * (250 - 50)
     */
    uint32_t ccr_value = (uint32_t)(50.0f + (angle * 200.0f / 180.0f));
    
    return ccr_value;
}

/**
 * @brief  驱动舵机执行目标动作
 */
void Servo_Set_Position(float target_angle) {
    uint32_t compare_val = Servo_Angle_To_Compare(target_angle);
    
    // 直接操作定时器通道的捕获比较寄存器 (CCR)，通过改变占空比重塑物理世界的时域信号
    __HAL_TIM_SET_COMPARE(&htim1, TIM_CHANNEL_1, compare_val);
}
