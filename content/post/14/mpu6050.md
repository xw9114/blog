## 技能概述

MPU6050 姿态解算的价值，不在于“读出一个角度”，而在于把陀螺仪的角速度、加速度计的重力投影、I2C 总线上的离散字节流，重新组织成对真实空间姿态的稳定估计。它广泛用于平衡车、云台、四旋翼、机械臂与运动捕捉设备，解决的核心痛点是：传感器原始数据天然带噪、陀螺积分必然漂移、加速度在动态场景下又会说“假话”，只有把零偏校准、采样时基与滤波融合同时做对，控制系统才有可靠的姿态基准。

## 核心底层概念解析

- **IMU 不是角度传感器**：MPU6050 本质上输出的是 **线加速度** 与 **角速度**，并不直接给出“俯仰角”或“横滚角”。所谓姿态，是软件在时域上对角速度积分、在空域上对重力方向投影后的估计结果。物理世界没有寄存器，寄存器只是对物理量的离散采样快照。
- **零偏（Bias）是积分系统的宿敌**：陀螺仪在静止时仍可能输出非零值，来源包括 **MEMS 结构应力、温漂、电源纹波、焊接应力**。角度计算满足 `angle = ∫omega dt`，所以哪怕只有 `0.5 dps` 的零偏，`60 s` 后也会累计出约 `30 deg` 漂移。对开环积分来说，偏置不是小误差，而是必然放大的系统性误差。
- **加速度计测到的是“比力”而非纯重力**：静止时，加速度计输出可以近似看作重力在机体坐标系中的投影；但一旦载体发生急加速、刹车或振动，读数里就会同时混入线加速度。也就是说，加速度计适合提供 **长期低频参考**，却不适合在剧烈运动时单独决定瞬时姿态。
- **I2C 连续突发读取比单寄存器轮询更可信**：MPU6050 的加速度、温度、陀螺数据在寄存器上是连续排布的。使用 **Burst Read** 一次性读取 `ACCEL_XOUT_H` 到 `GYRO_ZOUT_L` 的 14 字节，可以最大限度保证同一采样时刻的数据一致性，避免分多次读取时被新的采样周期“撕裂”。
- **姿态解算是坐标系映射问题**：传感器输出首先位于 **机体坐标系（Body Frame）**，而控制器想要的是相对于地面的 **参考坐标系（Earth Frame）**。`atan2` 的意义不是“套公式”，而是把三轴重力投影重新映射成可用于控制的角度描述。若安装方向、轴定义、正负号约定不统一，算法再漂亮也会输出错误姿态。
- **互补滤波的本质是频域分工**：陀螺仪短时间连续、抗振动，但长期漂移；加速度计长期稳定，却容易受瞬时线加速度污染。互补滤波并不是经验主义拼接，而是把 **高频变化交给陀螺积分，低频基准交给重力方向**。工程上，它对应的是一句很朴素的技术哲学：短时相信惯性，长时相信地心引力。
- **校准不是附属步骤，而是闭环系统的起点**：零偏校准、本体安装标定、采样周期约束、本振与电源完整性，决定了数字世界对物理世界的“翻译质量”。姿态估算看似是软件问题，实则是传感器、总线、时间基准和控制目标共同参与的系统工程。

## 代码能力展现

下面给出一个基于 STM32 HAL 的 MPU6050 姿态更新示例，覆盖初始化、陀螺零偏校准和互补滤波解算。代码刻意把“原始寄存器值如何映射为物理量”写在注释里，因为稳定的姿态估计从来不是 API 调用，而是尺度、时基与边界条件同时正确。

```c
#include "stm32f4xx_hal.h"
#include <math.h>
#include <string.h>

#define MPU6050_I2C_ADDR              (0x68U << 1)
#define MPU6050_REG_SMPLRT_DIV        0x19U
#define MPU6050_REG_CONFIG            0x1AU
#define MPU6050_REG_GYRO_CONFIG       0x1BU
#define MPU6050_REG_ACCEL_CONFIG      0x1CU
#define MPU6050_REG_PWR_MGMT_1        0x6BU
#define MPU6050_REG_ACCEL_XOUT_H      0x3BU

#define MPU6050_ACCEL_SENS_2G         16384.0f
#define MPU6050_GYRO_SENS_500DPS      65.5f
#define MPU6050_RAD_TO_DEG            57.2957795f
#define MPU6050_FILTER_TAU_S          0.25f

typedef struct
{
    float x;
    float y;
    float z;
} Mpu6050Vec3f_t;

typedef struct
{
    I2C_HandleTypeDef *hi2c;
    float pitch_deg;
    float roll_deg;
    Mpu6050Vec3f_t gyro_bias_dps;
    uint8_t bias_ready;
} Mpu6050Handle_t;

static float ClampF(float value, float min_value, float max_value)
{
    if (value < min_value)
    {
        return min_value;
    }

    if (value > max_value)
    {
        return max_value;
    }

    return value;
}

static float WrapDeg180(float angle_deg)
{
    while (angle_deg > 180.0f)
    {
        angle_deg -= 360.0f;
    }

    while (angle_deg < -180.0f)
    {
        angle_deg += 360.0f;
    }

    return angle_deg;
}

static int16_t JoinBytes(uint8_t msb, uint8_t lsb)
{
    return (int16_t)((uint16_t)(msb << 8) | lsb);
}

static HAL_StatusTypeDef MPU6050_WriteReg(Mpu6050Handle_t *imu, uint8_t reg, uint8_t value)
{
    return HAL_I2C_Mem_Write(imu->hi2c, MPU6050_I2C_ADDR, reg, I2C_MEMADD_SIZE_8BIT, &value, 1U, HAL_MAX_DELAY);
}

static HAL_StatusTypeDef MPU6050_ReadBurst(Mpu6050Handle_t *imu, uint8_t reg, uint8_t *buffer, uint16_t length)
{
    return HAL_I2C_Mem_Read(imu->hi2c, MPU6050_I2C_ADDR, reg, I2C_MEMADD_SIZE_8BIT, buffer, length, HAL_MAX_DELAY);
}

/**
 * @brief 初始化 MPU6050 为 200 Hz 输出、±2 g 加速度量程、±500 dps 陀螺量程。
 * @param imu MPU6050 句柄，内部包含 I2C 句柄与姿态状态。
 * @retval HAL_OK 初始化成功，其余值表示总线通信失败。
 */
HAL_StatusTypeDef MPU6050_Init(Mpu6050Handle_t *imu)
{
    if ((imu == NULL) || (imu->hi2c == NULL))
    {
        return HAL_ERROR;
    }

    memset(&imu->gyro_bias_dps, 0, sizeof(imu->gyro_bias_dps));
    imu->pitch_deg = 0.0f;
    imu->roll_deg = 0.0f;
    imu->bias_ready = 0U;

    /* 采样率公式：
     * Fs = Gyro_Output_Rate / (1 + SMPLRT_DIV)
     * 当 DLPF 使陀螺内部输出率为 1 kHz，且 SMPLRT_DIV = 4 时：
     * Fs = 1000 / (1 + 4) = 200 Hz
     */
    if (MPU6050_WriteReg(imu, MPU6050_REG_PWR_MGMT_1, 0x01U) != HAL_OK) return HAL_ERROR;
    if (MPU6050_WriteReg(imu, MPU6050_REG_CONFIG, 0x03U) != HAL_OK) return HAL_ERROR;       /* DLPF ≈ 44 Hz */
    if (MPU6050_WriteReg(imu, MPU6050_REG_SMPLRT_DIV, 0x04U) != HAL_OK) return HAL_ERROR;   /* 200 Hz */
    if (MPU6050_WriteReg(imu, MPU6050_REG_GYRO_CONFIG, 0x08U) != HAL_OK) return HAL_ERROR;  /* ±500 dps */
    if (MPU6050_WriteReg(imu, MPU6050_REG_ACCEL_CONFIG, 0x00U) != HAL_OK) return HAL_ERROR; /* ±2 g */

    return HAL_OK;
}

/**
 * @brief 在载体静止时估计陀螺零偏，结果以 dps 保存到句柄中。
 * @param imu MPU6050 句柄。
 * @param sample_count 参与平均的样本数，函数内部会限幅到 [128, 2000]。
 * @param sample_interval_ms 相邻采样间隔，函数内部会限幅到 [1, 10] ms。
 * @retval HAL_OK 校准成功，HAL_ERROR 表示参数错误或 I2C 读取失败。
 */
HAL_StatusTypeDef MPU6050_CalibrateGyroBias(Mpu6050Handle_t *imu, uint16_t sample_count, uint32_t sample_interval_ms)
{
    uint8_t raw[14];
    int32_t sum_x = 0;
    int32_t sum_y = 0;
    int32_t sum_z = 0;

    if ((imu == NULL) || (imu->hi2c == NULL))
    {
        return HAL_ERROR;
    }

    sample_count = (uint16_t)ClampF((float)sample_count, 128.0f, 2000.0f);
    sample_interval_ms = (uint32_t)ClampF((float)sample_interval_ms, 1.0f, 10.0f);

    for (uint16_t i = 0; i < sample_count; ++i)
    {
        if (MPU6050_ReadBurst(imu, MPU6050_REG_ACCEL_XOUT_H, raw, sizeof(raw)) != HAL_OK)
        {
            return HAL_ERROR;
        }

        /* 连续读取 14 字节，后 6 字节为陀螺三轴：
         * [8,9]  -> GYRO_XOUT
         * [10,11]-> GYRO_YOUT
         * [12,13]-> GYRO_ZOUT
         */
        sum_x += JoinBytes(raw[8], raw[9]);
        sum_y += JoinBytes(raw[10], raw[11]);
        sum_z += JoinBytes(raw[12], raw[13]);

        HAL_Delay(sample_interval_ms);
    }

    /* 线性映射公式：
     * gyro_dps = raw_gyro / 65.5
     * bias_dps = (Σ raw_gyro / N) / 65.5
     */
    imu->gyro_bias_dps.x = ((float)sum_x / (float)sample_count) / MPU6050_GYRO_SENS_500DPS;
    imu->gyro_bias_dps.y = ((float)sum_y / (float)sample_count) / MPU6050_GYRO_SENS_500DPS;
    imu->gyro_bias_dps.z = ((float)sum_z / (float)sample_count) / MPU6050_GYRO_SENS_500DPS;
    imu->bias_ready = 1U;

    return HAL_OK;
}

/**
 * @brief 读取原始数据并更新俯仰角、横滚角。
 * @param imu MPU6050 句柄，需先完成初始化与零偏校准。
 * @param dt_s 实际采样周期，单位 s；函数内部限幅到 [0.001, 0.02]。
 * @retval HAL_OK 更新成功，HAL_ERROR 表示参数错误、未校准或总线通信失败。
 */
HAL_StatusTypeDef MPU6050_UpdateAttitude(Mpu6050Handle_t *imu, float dt_s)
{
    uint8_t raw[14];
    float ax_g;
    float ay_g;
    float az_g;
    float gx_dps;
    float gy_dps;
    float accel_norm_g;
    float roll_acc_deg;
    float pitch_acc_deg;
    float roll_pred_deg;
    float pitch_pred_deg;
    float alpha;

    if ((imu == NULL) || (imu->hi2c == NULL) || (imu->bias_ready == 0U))
    {
        return HAL_ERROR;
    }

    dt_s = ClampF(dt_s, 0.001f, 0.02f);

    if (MPU6050_ReadBurst(imu, MPU6050_REG_ACCEL_XOUT_H, raw, sizeof(raw)) != HAL_OK)
    {
        return HAL_ERROR;
    }

    /* 原始值到物理量的线性映射：
     * accel_g = raw_acc / 16384.0    (±2 g)
     * gyro_dps = raw_gyro / 65.5     (±500 dps)
     */
    ax_g = (float)JoinBytes(raw[0], raw[1]) / MPU6050_ACCEL_SENS_2G;
    ay_g = (float)JoinBytes(raw[2], raw[3]) / MPU6050_ACCEL_SENS_2G;
    az_g = (float)JoinBytes(raw[4], raw[5]) / MPU6050_ACCEL_SENS_2G;

    gx_dps = ((float)JoinBytes(raw[8], raw[9]) / MPU6050_GYRO_SENS_500DPS) - imu->gyro_bias_dps.x;
    gy_dps = ((float)JoinBytes(raw[10], raw[11]) / MPU6050_GYRO_SENS_500DPS) - imu->gyro_bias_dps.y;

    /* 陀螺积分负责短时间连续性：
     * theta(k) = theta(k-1) + omega(k) * dt
     */
    roll_pred_deg = imu->roll_deg + gx_dps * dt_s;
    pitch_pred_deg = imu->pitch_deg + gy_dps * dt_s;

    accel_norm_g = sqrtf((ax_g * ax_g) + (ay_g * ay_g) + (az_g * az_g));

    /* 只有当合加速度接近 1 g 时，才认为“重力方向”足够可信。
     * 若此时载体正在剧烈加速或受强振动，直接使用加速度计会把线加速度误解释为姿态变化。
     */
    if ((accel_norm_g > 0.75f) && (accel_norm_g < 1.25f))
    {
        /* atan2 的意义是把机体坐标系中的重力投影重新映射为欧拉角。
         * roll  = atan2(ay, az)
         * pitch = atan2(-ax, sqrt(ay^2 + az^2))
         */
        roll_acc_deg = atan2f(ay_g, az_g) * MPU6050_RAD_TO_DEG;
        pitch_acc_deg = atan2f(-ax_g, sqrtf((ay_g * ay_g) + (az_g * az_g))) * MPU6050_RAD_TO_DEG;

        /* 离散互补滤波：
         * angle_k = alpha * (angle_{k-1} + omega_k * dt) + (1 - alpha) * angle_acc
         * alpha   = tau / (tau + dt)
         * tau 越大，系统越信任陀螺短时连续性；tau 越小，系统越依赖重力的长期约束。
         */
        alpha = MPU6050_FILTER_TAU_S / (MPU6050_FILTER_TAU_S + dt_s);

        imu->roll_deg = (alpha * roll_pred_deg) + ((1.0f - alpha) * roll_acc_deg);
        imu->pitch_deg = (alpha * pitch_pred_deg) + ((1.0f - alpha) * pitch_acc_deg);
    }
    else
    {
        /* 动态冲击期间暂停加速度校正，避免把外部线加速度当作姿态变化。 */
        imu->roll_deg = roll_pred_deg;
        imu->pitch_deg = pitch_pred_deg;
    }

    /* 边界保护：
     * 1. pitch 限幅到接近 ±90 deg，避免极端姿态下控制器获得非物理突变值。
     * 2. roll 包裹到 [-180, 180]，便于上层控制与显示逻辑统一处理。
     */
    imu->pitch_deg = ClampF(imu->pitch_deg, -89.9f, 89.9f);
    imu->roll_deg = WrapDeg180(imu->roll_deg);

    return HAL_OK;
}
```

这段实现的关键，不是把公式“写出来”，而是把物理条件翻译成代码约束：静止时先做零偏校准，动态时谨慎使用重力校正，采样周期必须真实可信，最终让姿态估计成为控制闭环里可被信任的一环，而不是噪声与漂移的放大器。
