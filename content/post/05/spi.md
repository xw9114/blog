## 技能概述

作为嵌入式底层驱动开发的核心能力，我深度掌握 **SPI (Serial Peripheral Interface)** 通讯协议。能够熟练驾驭从软件模拟时序（Bit-banging）到硬件 SPI 控制器的深度配置。在实际工程中，成功驱动过 **TFT/OLED 屏幕**（高帧率刷新）、**W25Qxx 系列 Flash**（海量数据存储）及各类高速传感器。具备解决高速信号串扰、多设备总线复用及 DMA 协同的高级调试与架构设计能力。

## 核心概念白话解读：秒懂 SPI 的“霸道总裁与专属内线”模型

对比 I2C 的“大课堂喊话点名”，SPI 更像是公司里**“霸道总裁与员工的高效办公网络”**，主打一个简单粗暴、高速双向：

* **主机 (Master)**：公司总裁。整个通讯的节奏、什么时候开会，全由总裁掌控。
* **从机 (Slave)**：员工（如 Flash、屏幕）。只能听总裁的指令，不能主动找总裁。
* **SCK (时钟线)**：总裁桌上的节拍器。总裁打一个拍子，双方就强制交换一次信息。
* **MOSI (Master Out Slave In)**：总裁对员工下达指令的专属麦克风。
* **MISO (Master In Slave Out)**：员工向总裁汇报工作的专属麦克风。因为有两根独立的线，他们可以**同时说话（全双工）**。
* **CS/NSS (片选线)**：总裁桌上的**专属内线电话**。总裁想找哪个员工，不浪费时间喊名字，直接按下对应员工的电话专线（拉低该从机的 CS 引脚）。这种“一对一硬连线”方式免去了地址发送，是 SPI 速度极快的原因之一。

## 核心能力矩阵与“潜规则”

在实际工程项目中，驾驭 SPI 的核心在于掌控以下几个关键“潜规则”：

* **四大工作模式的精准匹配 (CPOL & CPHA)**：熟知时钟极性（CPOL）和时钟相位（CPHA）的排列组合。深刻理解“空闲时钟电平”与“奇/偶数边沿采样”的映射关系。拿到任何芯片手册，都能瞬间识别时序图并匹配正确的 Mode 0~3（例如 W25Q 系列通常严格要求 Mode 0 或 Mode 3）。
* **全双工与“移位寄存器”本质**：洞悉 SPI 硬件底层的实质是**“两个环形连接的移位寄存器”**。深刻理解“发一个字节的同时必然收一个字节”的强制绑定逻辑。熟练运用“发送 Dummy Data（哑数据如 0xFF）来套取从机有效数据”的实战技巧。
* **推挽输出与高频信号完整性**：对比 I2C 的开漏输出，SPI 引脚采用**推挽输出 (Push-Pull)**，驱动能力极强，轻松达到数十兆赫兹 (MHz) 速率。在多从机共享 MISO 线时，能精准处理失活从机 MISO 引脚的**高阻态 (Hi-Z)** 配置问题，避免总线电平冲突与短路烧毁。

## 代码能力展现

以下为基于 C 语言编写的软件模拟 SPI **Mode 0 (CPOL=0, CPHA=0)** 的双向收发底层实现。展示了对“时钟边沿触发”和“全双工位移”逻辑的极致把控：

```c
/**
 * @brief 软件模拟 SPI 收发一个字节 (Mode 0: 空闲低电平，第一个上升沿采样)
 * @param txData: 主机待发送的数据 (MOSI)
 * @return uint8_t: 主机接收到的数据 (MISO)
 */
uint8_t SPI_ReadWriteByte(uint8_t txData) {
    uint8_t rxData = 0;
    uint8_t i;

    for (i = 0; i < 8; i++) {
        // 1. 数据准备阶段：主机把数据放到 MOSI 线上 (最高位 MSB 先行)
        // 此时 SCK 保持在低电平 (CPOL=0 的空闲状态或上一个周期的低电平)
        if (txData & 0x80) {
            SPI_MOSI_HIGH();
        } else {
            SPI_MOSI_LOW();
        }
        txData <<= 1;
        SPI_Delay(); // 满足数据建立时间 (Setup Time)

        // 2. 产生时钟上升沿 (第一个边沿)：
        // 此时从机自动捕获 MOSI 数据，同时主机读取 MISO 上的从机数据
        SPI_SCK_HIGH();
        rxData <<= 1;
        if (SPI_MISO_READ()) { // 采样从机通过 MISO 传来的位
            rxData |= 0x01;
        }
        SPI_Delay(); // 满足数据保持时间 (Hold Time)

        // 3. 产生时钟下降沿 (第二个边沿)：
        // 时钟回落，通知双方准备切换下一位数据
        SPI_SCK_LOW();
        SPI_Delay();
    }

    // 核心体现：全双工特性。发完 8 bit，同时也收满了 8 bit。
    return rxData; 
}
