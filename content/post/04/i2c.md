## 技能概述

作为嵌入式开发的核心技能，我深度掌握 **I2C (Inter-Integrated Circuit)** 通讯协议。能够熟练处理从底层 GPIO 软件模拟时序到硬件控制器（I2C Peripheral）的配置开发。在实际项目中，如 **AT24C02 (EEPROM)** 数据存储、**MPU6050** 姿态传感器读取等场景下，均能实现稳定、高效的数据交互。

## 核心概念白话解读：秒懂 I2C 的“课堂模型”

在死记硬背枯燥的时序图之前，我通常这样在脑海中构建 I2C 的运行逻辑：**把 I2C 总线想象成一个课堂。**

* **主机（Master / 比如单片机）**：课堂上的**老师**。
* **从机（Slave / 比如传感器、EEPROM）**：课堂里的**学生**。所有学生都挂在同一条线上。
* **SCL（时钟线）**：老师打拍子的手。老师拍一下手，大家就传一个纸条。（控制节奏的永远是老师）
* **SDA（数据线）**：传纸条的唯一通道。

**一次完整的数据读取过程，就是一次“老师点名收作业”：**

1.  **起始信号 (Start)**：老师猛拍一下讲桌：“大家安静，我要点名了！”（表现为：SCL 为高电平时，SDA 从高变低）。
2.  **发送地址 (Address)**：老师喊：“学号为 1010000 的同学！”（主机连发 7 个 Bit 的设备地址）。
3.  **读写控制 (R/W)**：老师紧接着说：“把你的作业本交上来！”（读：1）。
4.  **应答信号 (ACK)**：被叫到的那个学生立刻举手喊：“到！”（从机强行把 SDA 拉低一个时钟周期）。
5.  **传输数据 (Data)**：两人开始传纸条（发送或接收 8 个 Bit 的数据）。传完 8 个 Bit，接收方又要喊一次“收到 (ACK)”。
6.  **停止信号 (Stop)**：老师说：“好，这件事办完了，大家自由活动。”（表现为：SCL 为高电平时，SDA 从低变高）。

## 核心能力矩阵与“潜规则”

除了生动的概念映射，我更关注 I2C 在实际工程中的硬件与时序“潜规则”：

- **硬件基础：线与逻辑与开漏输出 (Open-Drain)**
  I2C 设备的引脚只能拉低，不能主动拉高，必须依靠外部**上拉电阻**（如 4.7kΩ）。这种设计完美实现了“线与”逻辑：只要有一个设备拉低，总线就是低电平。这既防止了多设备同时发 0 和 1 导致的短路烧毁，又是多主机冲突检测的基础。
- **时序精准控制：何时能动，何时不许动**
  精通 SCL 与 SDA 的配合准则：**SCL 为高电平时**，数据线 SDA 必须保持稳定（老师盯着看，不许乱动，准备读取）；**SCL 为低电平时**，数据线 SDA 才允许改变电平（老师闭上眼，准备换下一张纸条）。唯一例外是 Start 和 Stop 信号。

## 代码能力展现

以下为基于 C 语言编写的软件模拟 I2C 发送单字节的经典实现，展示了对“老师打拍子”时序微秒级控制的理解：

```c
/**
 * @brief I2C 发送一个字节数据 (Master Write)
 * @param byte: 待发送的 8-bit 数据
 * @return uint8_t: 返回从机应答信号 (0: ACK, 1: NACK)
 */
uint8_t I2C_SendByte(uint8_t byte) {
    uint8_t i;
    
    for (i = 0; i < 8; i++) {
        // 1. SCL 为低电平 (老师闭眼)，允许 SDA 改变电平准备数据
        I2C_SCL_LOW(); 
        if (byte & 0x80) { // 从最高位开始传输 (MSB)
            I2C_SDA_HIGH();
        } else {
            I2C_SDA_LOW();
        }
        byte <<= 1;
        I2C_Delay(); // 确保建立时间 (Setup Time)

        // 2. SCL 拉高 (老师睁眼盯着看)，通知从机读取稳定数据
        I2C_SCL_HIGH();
        I2C_Delay(); // 确保保持时间 (Hold Time)
    }

    // 3. 接收从机应答位 (ACK / 学生喊“到！”)
    I2C_SCL_LOW();
    I2C_SDA_INPUT_MODE(); // 切换 SDA 为输入模式，释放总线给从机用
    I2C_Delay();
    I2C_SCL_HIGH();
    
    uint8_t ack = I2C_SDA_READ(); // 读取从机拉低的应答信号
    I2C_Delay();
    I2C_SCL_LOW();
    
    return ack; // 返回 0 表示有人应答，返回 1 表示无应答
}
